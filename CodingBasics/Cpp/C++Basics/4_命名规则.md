# 命名风格（根据类型分类）

##  变量名（变量/参数/成员变量）

- 使用**小写字母 + 下划线分隔**（snake_case）
- 成员变量通常加前缀 `m_` 或 `_`（取决于风格）

```c++
int student_id;
double temperature_celsius;
std::string m_user_name; // 成员变量（前缀）
```

## 函数名

* 使用**驼峰命名（camelCase）或动词加描述**

```c++
void calculateSum();
bool isPrime(int number);
```

## 类名 / 结构体名

- 使用**首字母大写的驼峰命名（PascalCase）**

```c++
class BinaryTree;
struct HttpRequest;
```

## 常量名

- 使用**全大写 + 下划线（SNAKE_CASE）**

```c++
const double PI = 3.14159;
#define MAX_BUFFER_SIZE 1024
```

## 枚举值

- 推荐使用类枚举（C++11）+ PascalCase 成员名

```
cpp复制编辑enum class Color {
    Red,
    Green,
    Blue
};
```

# 常见前缀/后缀规范（可选但推荐）

| 类型               | 示例                      | 说明             |
| ------------------ | ------------------------- | ---------------- |
| `is`、`has`、`can` | `isValid`, `hasError`     | 返回布尔值的函数 |
| `_ptr`             | `image_ptr`               | 指针变量         |
| `_list`, `_map`    | `user_list`, `config_map` | 容器变量         |
| `_count`, `_size`  | `error_count`             | 数值型表示量     |

# 文件命名规范

| 类型     | 示例                                    |
| -------- | --------------------------------------- |
| 头文件   | `matrix_utils.h`, `config_parser.hpp`   |
| 实现文件 | `matrix_utils.cpp`, `config_parser.cpp` |
| 类文件   | `Student.cpp`, `Student.h`              |

| 文件类型                    | 定义与用途                                                   |
| --------------------------- | ------------------------------------------------------------ |
| **头文件（Header File）**   | 文件后缀为 `.h` 或 `.hpp`，用于**声明**函数、类、常量、宏等接口，不包含完整实现。供其他 `.cpp` 文件通过 `#include` 引入。 |
| **实现文件（Source File）** | 文件后缀为 `.cpp`，用于实现头文件中声明的函数、类等功能。一般不会被其他 `.cpp` 文件直接包含，而是由编译器编译。 |
| **类文件（Class File）**    | 并不是一种文件格式，而是一种**组织方式**：即为一个类分别创建 `.h` 和 `.cpp` 文件，一起构成“类文件”。 |

# 命名中的禁忌与注意事项

| 不推荐做法                          | 原因                           |
| ----------------------------------- | ------------------------------ |
| 单字符命名（除循环变量如 `i`, `j`） | 可读性差                       |
| 使用保留字或标准库名                | 易出错                         |
| 过度缩写                            | 除非众所周知（如 `ID`, `URL`） |
| `l` 和 `O` 等难辨字符               | 阅读歧义性                     |
