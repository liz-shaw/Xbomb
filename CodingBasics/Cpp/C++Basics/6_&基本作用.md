**引用（reference）**

------

## ✅ 一、`vector<int>& a` 是什么意思？

它的意思是：

> `a` 是对某个 `vector<int>` 对象的 **引用**，而**不是一份拷贝**。

------

## ✅ 二、& 的基本作用：**声明引用变量**

```cpp
int x = 10;
int& y = x; // y 是 x 的别名，y 和 x 绑定在一起

y = 20;     // 改变 y 就是改变 x
cout << x;  // 输出 20
```

对比：

```cpp
int x = 10;
int y = x; // 拷贝 x 的值

y = 20;
cout << x; // 输出 10（x 不变）
```

------

## ✅ 三、在函数参数中使用 `&` 的意义（关键）

```cpp
bool isSubset(vector<int>& a, vector<int>& b)
```

这表示：

- `a` 和 `b` 是传进来的原始 vector 的引用；
- 在函数内部修改 `a`, `b`，会影响到外部调用者的变量；
- 不会发生 **值拷贝**，避免内存浪费和性能损失。

### 对比（传值 vs 引用）：

| 写法                   | 含义               | 优点               |
| ---------------------- | ------------------ | ------------------ |
| `vector<int> a`        | 值传递（复制一份） | 安全但效率低       |
| `vector<int>& a`       | 引用传递           | 高效、可修改原数据 |
| `const vector<int>& a` | 常量引用（只读）   | 高效、保护原数据   |

------

## ✅ 四、一个实际例子演示

```cpp
#include <iostream>
#include <vector>
using namespace std;

void modify(vector<int>& v) {
    v.push_back(100); // 修改原容器
}

int main() {
    vector<int> data = {1, 2, 3};
    modify(data);

    for (int x : data) {
        cout << x << " ";  // 输出：1 2 3 100
    }
}
```

如果 `modify(vector<int> v)` 是值传递，则原容器不变。

------

## ✅ 五、小结：什么时候用引用？

| 场景                        | 是否用引用？  | 建议写法        |
| --------------------------- | ------------- | --------------- |
| 大对象（如 vector, string） | ✅             | `const &` / `&` |
| 小对象（int, char）         | ❌（传值更快） | 直接传值即可    |
| 需要修改原始对象            | ✅             | `&`             |
| 不希望修改，只读传入        | ✅             | `const &`       |
